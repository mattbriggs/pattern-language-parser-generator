    Roadmap¶

    This roadmap describes planned improvements to Pattern Language Miner, organised by release milestone. Items are subject to change based on user feedback and community contributions.

    v1.1 — Multilingual & Quality Improvements¶

    Target: Q2 2025

    Pattern Quality¶

    • Replace heuristic infer_problem_from_solution with an NLU model (e.g. spaCy transformer pipeline)
    • Add stopword filtering per language to reduce noise in extracted n-grams
    • Support spaCy as an alternative tokeniser to NLTK

    Multilingual Support¶

    • Test and document corpus behaviour for French, German, and Spanish
    • Make sentence-transformer model configurable per language in config.yaml

    Developer Experience¶

    • Add a --dry-run flag to analyze that reports how many patterns would be extracted without writing files
    • Publish package to PyPI

    v1.2 — Interactive Review & Editing¶

    Target: Q3 2025

    Web UI¶

    • Streamlit-based pattern review interface — browse, edit, approve, and reject extracted patterns
    • In-browser cluster visualisation with hover tooltips

    Pattern Management¶

    • Pattern deduplication: merge patterns with cosine similarity > threshold
    • Pattern versioning: track changes to enriched fields over time
    • CRUD REST API for pattern library management

    v1.3 — Pipeline & Automation¶

    Target: Q4 2025

    Pipeline Enhancements¶

    • Implement Pipeline orchestrator integration for the CLI (run all steps in one command via pattern-miner run-all)
    • --watch mode: re-run analyze automatically when input files change
    • Parallel document processing using concurrent.futures

    CI/CD Integration¶

    • GitHub Actions workflow template for automated pipeline runs
    • Pre-commit hook to validate config.yaml on commit
    • Docker image published to GitHub Container Registry

    v2.0 — Semantic Enrichment & LLM Integration¶

    Target: H1 2026

    LLM-Powered Enrichment¶

    • Optional GPT-4 / Claude enrichment backend for high-quality summary, problem, and rationale inference
    • Configurable enrichment strategy selection in config.yaml

    Advanced Clustering¶

    • Hierarchical clustering as an alternative to KMeans
    • Automatic cluster-count selection using silhouette scoring
    • Named cluster labels generated by LLM summarisation

    Pattern Schema¶

    • Extended schema v2 with rationale, warning, related_standards, and type fields
    • Schema migration tool for upgrading existing pattern libraries

    Backlog (Unscheduled)¶

    • PDF document parsing support
    • Confluence and Notion page ingestion connectors
    • Obsidian vault integration
    • Pattern-to-template export (Jinja2, Markdown templates)
    • Visual pattern-map editor
    • Fine-tuned sentence-transformer model for technical documentation

    Completed¶

    See the Changelog for a full list of delivered features.
